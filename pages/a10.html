💥 Manejo Inadecuado de Errores y Excepciones

● Riesgo: MEDIO

Cuando ocurre un error en la aplicación y no se maneja correctamente, el servidor puede mostrar al usuario información técnica interna: rutas de archivos, contraseñas en la configuración, versiones de software o la estructura interna del código.

Demo Interactiva
https://demo-app.ejemplo.com/a10_2025

Provoca un error en el servidor y observa cuánta información sensible revela la aplicación vulnerable.

✅ El usuario ve un mensaje genérico. Los detalles técnicos completos se guardan solo en los logs internos del servidor.
🙈 Mensajes genéricos al usuario — "Ha ocurrido un error. Referencia: ERR-2025-001" (sin detalles técnicos)
📋 Logs internos completos — El stack trace completo se guarda de forma segura, nunca se muestra al usuario
🔍 ID de correlación — Permite al equipo técnico encontrar el error exacto en los logs usando el código de referencia
🚫 Deshabilitar debug en producción — El modo debug nunca debe estar activo en un servidor real
⚠️ Solo para fines educativos — No usar en producción
