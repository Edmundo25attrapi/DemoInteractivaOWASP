🧩 Fallos de Integridad de Software y Datos

● Riesgo: ALTO

La aplicación usa código o datos externos (librerías de CDN, actualizaciones automáticas, archivos descargados) sin verificar que no hayan sido alterados. Un atacante puede reemplazar ese código con su propia versión maliciosa.

Demo Interactiva
https://demo-app.ejemplo.com/a08_2025

Simula cargar una librería desde un CDN externo. Si el CDN es comprometido y no verificamos la integridad, el código malicioso se ejecuta en el navegador de todos los usuarios.

✅ Subresource Integrity (SRI) es un mecanismo del navegador que verifica que los archivos externos no hayan sido modificados.
🔏 SRI (Subresource Integrity) — Añade integrity="sha384-..." a todos los scripts y estilos externos
🔄 Firma de artefactos CI/CD — Firma digitalmente cada build para detectar modificaciones en el pipeline
🚫 Evitar deserialización insegura — Nunca deserializar datos no confiables (pickle en Python, Java ObjectStream)
🔍 Análisis de código en pipeline — Escaneo estático automático en cada pull request
⚠️ Solo para fines educativos — No usar en producción
